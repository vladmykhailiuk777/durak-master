# DurakMaster — Консольна гра «Підкидний Дурень»

Консольна реалізація класичної карткової гри «Підкидний Дурень» на мові C++ із
використанням принципів об'єктно-орієнтованого програмування (ООП).
Проєкт має гнучку архітектуру, систему розумних підказок для гравців та
адаптований інтерфейс користувача.

## Як зібрати і запустити

Проєкт написаний на стандарті C++17 і не потребує сторонніх бібліотек. Потрібен лише компілятор `g++` або середовище Visual Studio.

**g++ (Linux / macOS / MinGW на Windows):**

```bash
g++ -std=c++17 main.cpp -o durak_master
./durak_master

```

**Visual Studio:**

1. Створи новий проєкт *Консольний додаток C++ (Console App)*.
2. Додай вміст файлу з кодом у `main.cpp`.
3. Переконайся, що в налаштуваннях проєкту вибрано стандарт мови **C++17** або новіший.
4. Натисни **F5** для запуску.

## Що вміє програма

* **Гнучкі налаштування сесії:** підтримка динамічної кількості гравців (від 2 до 6) та вибір розміру колоди (24, 36 або 52 карти).
* **Кастомізація перед грою:** можливість введення унікальних імен для кожного гравця на початку партії.
* **Інтелектуальна система підказок:** під час фази підкидання програма автоматично аналізує карти сусіда, виводить лише дозволені варіанти для ходу або автоматично пропускає хід, якщо підкинути нічого.
* **Автоматичне кросплатформне очищення консолі:** екран вчасно очищується після кожного раунду («Бито» чи «Взято»), що робить ігрове поле охайним і читабельним.
* **Повноцінний вбудований довідник:** детально розписані правила гри безпосередньо всередині консольного меню.
* **Захист від некоректного введення:** програма захищена від випадкових натискань невірних символів чи літер (буфер введення автоматично очищується).

## Структура класів (ООП архітектура)

Проєкт побудовано за канонами ООП, де кожен ігровий елемент винесено в окрему сутність:

| Клас | За що відповідає |
| --- | --- |
| `Card` | **Карта:** зберігає масть, ранг та статус козиря. Має перевантажені оператори `<` та `==`, а також метод `canBeat()` для перевірки правил відбивання. |
| `Deck` | **Колода:** динамічно генерує карти залежно від обраного розміру, автоматично визначає козир по останній карті та тасує карти за допомогою `std::shuffle` та `std::chrono`. |
| `Player` | **Абстрактний базовий клас гравця:** містить вектор карт у руці, ім'я та віртуальні методи для вибору дій. |
| `HumanPlayer` | **Конкретний клас гравця (Користувач):** реалізує методи взаємодії користувача з грою через консольне введення. |
| `Table` | **Ігровий стіл:** контролює логіку карт в атаці та захисті, перевіряє можливість підкинути карту певного рангу. |
| `GameMode` | **Інтерфейс режиму гри:** базовий клас для керування логікою ходів. |
| `ThrowingDurak` | **Підкидний режим:** спадкує `GameMode`. Містить головний алгоритм раунду: черговість ходів лівого/правого сусідів, фіксацію відбою або взяття карт. |
| `GameSession` | **Ігровий менеджер:** координує всю гру. Роздає карти порівну після раунду, відстежує вихід гравців із гри та оголошує фінального програвшого. |

### Ієрархія сутностей:

```text
GameSession (Керує процесом)
 ├── Deck (Керує картами) ──> Card
 ├── Table (Керує полем)  ──> Card
 ├── GameMode
 │     └── ThrowingDurak (Логіка підкидного раунду)
 └── Player (Abstract)
       └── HumanPlayer (Живий користувач)

```
